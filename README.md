# 🌿 Revive — Spaced Repetition Study Tracker

Revive turns any lesson, note, or document into a scheduled revision plan. Instead of cramming, it uses spaced-repetition intervals based on the forgetting curve (1 day → 3 days → 10 days → 30 days → 90 days) to tell you exactly when to review something — right before you'd otherwise forget it.

It's built as a self-contained front-end app (no backend required to run the demo), with optional automation hooks for real email reminders and AI-generated note summaries.

## ✨ Features

- **Spaced Repetition Scheduling** — Revisions are automatically scheduled at Immediate, 3-Day, and 10-Day intervals based on the Ebbinghaus forgetting curve
- **AI-Powered Study Summaries** — Upload your notes (PDF, DOCX, or TXT) and get a concise bullet-point revision summary generated by Claude AI
- **Email Notifications** — Receive styled HTML emails when a lesson is added and when each revision is due, including your AI summary
- **Audio / Beep Reminders** — Schedule in-browser audio alerts for upcoming revision sessions
- **Retention Tracking** — Mark revisions as "Remembered" or "Forgot" to dynamically update your retention score per lesson
- **Demo Mode** — Test the full revision flow with compressed intervals (minutes instead of days)
- **Automation Log** — A live activity log tracks every scheduled revision, email sent, upload, and AI generation
- **n8n Workflow Integration** — Export a ready-to-import n8n automation workflow via `/api/n8n/workflow`

--

## 🧠 The Science

Revive's scheduling is built around two ideas:

1. **The Ebbinghaus Forgetting Curve** — without review, memory retention drops to roughly 40% within 24 hours and under 20% within a week.
2. **Spaced Repetition** — each timely revision flattens the curve, extending the interval before the next review is needed.

## 🚀 Getting Started

### Prerequisites

- Python 3.8+
- A Gmail account with an [App Password](https://support.google.com/accounts/answer/185833) (for email features)
- An [Anthropic API key](https://console.anthropic.com/) (for AI summaries)

### Installation

**1. Clone the repository**
```bash
git clone https://github.com/ernandhinni/revive.git
cd revive
```

**2. Install dependencies**
```bash
pip install -r requirements.txt
```

**3. Run the app**
```bash
python app.py
```

**4. Open in your browser**
```
http://localhost:5000
```

## ⚙️ Configuration

All configuration is done through the app's Settings panel, or directly via API.

### Email (SMTP)
```http
POST /api/config/email
Content-Type: application/json

{
  "host": "smtp.gmail.com",
  "port": 587,
  "email": "you@gmail.com",
  "password": "your-app-password"
}
```

### Anthropic API Key (for AI Summaries)
```http
POST /api/config/ai
Content-Type: application/json

{
  "anthropic_key": "sk-ant-..."
}
```

Alternatively, set the environment variable before starting the server:
```bash
export ANTHROPIC_API_KEY=sk-ant-...
python app.py
```

## 🔄 How Spaced Repetition Works in Revive

When you add a lesson, Revive immediately schedules three revisions:

```
Lesson Added
     │
     ├── ✅ Immediate  →  right now
     ├── 📅 3-Day      →  3 days later
     └── 📆 10-Day     →  10 days later
```

A background thread checks every 30 seconds for due revisions and fires email notifications automatically. When you complete a revision:

- **Remembered** → retention score goes up (+20)
- **Forgot** → retention score goes down (−30) and a retry revision is scheduled

## 🤖 AI Summary Feature

When you upload notes for a lesson (PDF, DOCX, or TXT), Revive:

1. Extracts the text from the file
2. Sends it to the Claude API with a study-focused prompt
3. Returns a structured summary with an overview, 5–8 key bullet points, and a "remember this" tip
4. Attaches the summary to all future revision reminder emails

## 🔒 Security Notes

- Passwords are hashed with SHA-256 before storage
- Auth tokens are regenerated on every login
- API keys and SMTP credentials are stored in `database.json` — **do not commit this file to version control**
- Add `database.json` and `uploads/` to your `.gitignore`

```gitignore
database.json
uploads/
__pycache__/
*.pyc
.env
```
